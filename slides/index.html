Jasmine

A behavior driven development framework for testing JavaScript

  • Tests non DOM JavaScript code
  • A test suite is made of one or more specs.
  • A spec contains one or more expectations that evaluate to true or false. 
    describe("A suite", function() {
      it("contains spec with an expectation", function() {
        expect(true).toBe(true);
      });
    });

Jasmine can be run via node in a console, or as JavaScript that outputs results in a browser

Installing and Running Jasmine with nodeJs

  • open Terminal, then run:
  • npm install jasmine-node -g
    • Fetches jasmine-node files from npmjs.org
    • Run and re-install for any mkdir errors: sudo chown -R $USER /usr/local
  • cd to root folder of your project
  • To run test specs: jasmine-node spec/
    • spec directory contains Jasmine tests files
    • For error File: 'spec/ is missing' run mkdir spec directory in project root then run jasmine-node spec/

Running Jasmine as standalone JavaScript

  • Download the files from GitHub (https://github.com/pivotal/jasmine/downloads)
  • Place uncompressed files at root of project
  • SpecRunner.html displays Jasmine test results for non-DOM specs
  • taskAppSpecRunner.html displays Jasmine-jQuery test results for DOM specs
